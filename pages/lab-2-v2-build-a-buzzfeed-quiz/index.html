Minimal Functional Specs

  • Quiz Page:
    • Questions:
      • display several questions (enough for this to be interesting)
      • have a header image and some text
      • have some number of potential answers
    • Question Answers:
      • be either text or an image (demonstrate some of both)
      • have 4 potential display states:
        • initial none selected state
        • have a :hover effect
        • have a clicked/selected state
        • when one answer is chosen all other answers should change to a not-selected state
      • have an associated value to be used to calculate the overall quiz output
    • Done Button:
      • calculate the quiz output
    • Output Display:
      • should have text and an image
      • should not show unless the calculations are finished
      • should display an error if not all questions were answered

HTML Tips: Radio Buttons

One thing you may have noticed is that when you select an answer it is an exclusive selection. There is an input type that already does that automatically. Yup, radio boxes!

But the default styling of radio boxes is so ugly! Sure, but you can hide the actual radio box and use it for display logic.

Here is the basic syntax for a radio button with an image label:

HTML
12
<label for="puppy_question"><img src="someimage.png" /></label>
<input type="radio" id="puppy_question" value="puppy" />
CSS
123
input[type='radio'] {
  display: none;
}

You can nest the input inside of the label and then you don't have to do the for="elementID" part:

HTML
1234
<label>
  <img src="someimage.png" />
  <input type="radio" value="puppy" />
</label>

Important: The way you group radio buttons together (clicking one de-selects the others) is by their name property. Same name = same group.

These two can both be selected (different groups):

HTML
12345678
<label>
  <img src="dog.png" />
  <input type="radio" name="favorite_dog" value="dalmation" />
</label>
<label>
  <img src="cat.png" />
  <input type="radio" name="favorite_cat" value="tiger" />
</label>

These two will auto-unselect each other (same group):

HTML
12345678
<label>
  <img src="dalmation.png" />
  <input type="radio" name="favorite_dog" value="dalmation" />
</label>
<label>
  <img src="retriever.png" />
  <input type="radio" name="favorite_dog" value="retriever" />
</label>

JavaScript: Loading Data with Fetch

You'll need to load the JSON data using fetch. Here's the basic pattern:

JavaScript
123456
fetch('quiz.json')
  .then(response => response.json())
  .then(data => {
    // Now you can work with data
    console.log(data.title);
  });

To gather all selected radio button values when the user clicks submit:

JavaScript
12345678
document.getElementById('submit-btn').addEventListener('click', () => {
  const checked = document.querySelectorAll("input[type='radio']:checked");
  const choices = Array.from(checked).map(radio => radio.value);

  // choices is now an array like ["python", "javascript", "python"]
  // Calculate the most common outcome
  console.log(choices);
});

Some useful vanilla JS methods:

  • element.style.display = 'none' - hide an element
  • element.style.display = 'block' - show an element
  • element.textContent = 'new text' - change text
  • element.innerHTML = '<p>html</p>' - change HTML content

When Things Break

JavaScript timing can be tricky. You might encounter situations where:

  • Data seems to be missing when you try to use it
  • Elements can't be found even though they're in your HTML
  • Click handlers behave unexpectedly

When this happens:

  1. Open the browser DevTools console (F12 or Cmd+Option+I)
  2. Look for error messages
  3. Add console.log() statements to trace execution order
  4. Ask Claude questions like "Why might this variable be undefined here?"

Learning Goals

By the end of this lab, you should understand:

  • Async execution: Why JavaScript doesn't always run top-to-bottom
  • Promises & fetch: How to properly wait for data to load
  • DOM ready: When it's safe to access HTML elements
  • Debugging: How to use console.log and DevTools to trace issues
