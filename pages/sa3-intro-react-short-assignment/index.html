Choose Your API

For this assignment, you'll pick an API to build your search app around. The tutorial uses YouTube as the example, but you are encouraged to choose something different! Here are some great options:

No API Key Required

These APIs are the easiest to get started with - no signup needed:

  • iTunes Search API - Search music, movies, podcasts, audiobooks. Returns album art, previews, and descriptions. ~20 requests/minute limit.
  • Open Library API - Search millions of books. Returns covers, authors, publish dates, and descriptions.
  • PokeAPI - All Pokemon data you could want. Images, stats, abilities, evolutions. Fun and well-documented.
  • Rick and Morty API - Characters, locations, and episodes from the show. Great images and data. 10,000 requests/day.
  • Art Institute of Chicago API - Search 50,000+ artworks. High-quality images, artist info, and descriptions. All public domain.
  • Dog CEO API - 20,000+ images across 120+ dog breeds. Random dogs, breed-specific searches, and sub-breeds.
  • NHL Web API - Official (unofficial) hockey API. Teams, players, stats, schedules. No key needed.
  • Open-Meteo - Weather forecasts, temperature, precipitation, wind. Location-based (not search-based) - a different but rewarding challenge. No key needed.

Simple API Key (Free Tier)

These require a quick signup to get an API key, but have generous free tiers:

  • YouTube Data API - The example used in this tutorial. Search videos, get thumbnails, embed players.
  • Giphy API - Search GIFs and stickers. Easy to embed, fun results. 100 requests/hour on free tier.
  • TMDB (The Movie Database) - Search movies and TV shows. Posters, cast, ratings, trailers. Very rich data.
  • Pexels API - Beautiful stock photos and videos. 200 requests/hour, 20,000/month free.
  • Pixabay API - 1.9 million free photos, illustrations, and videos. Unlimited requests.
  • NASA APIs - Astronomy Picture of the Day, Mars rover photos, Earth imagery. Educational and stunning visuals.
  • Vimeo API - Search and embed videos. Similar to YouTube but different content library. OAuth 2.0 authentication.
  • The Cat API - Thousands of cat images, breed info, and cat facts. 10,000 requests/month free.

Pick one that interests you! The core React concepts (components, state, props, effects) work the same regardless of which API you choose. The tutorial will walk through YouTube, but you should adapt the code to your chosen API.

useEffect Hook

Enter the useEffect Hook, perhaps one of the most misunderstood hooks. The useEffect hook allows you to synchronize a component with an external system. In this case we want to synchronize with a list returned by a query to the youtube api.

  • Learn about useEffect

    The useEffect hook is very powerful and essential hook, but is also very weird. In essence you are declaring that your React component, and specifically your render, has a side effect dependency. It needs to synchronize with something external, often an api.

    If you have a button that triggers an event that changes state - that does NOT need useEffect. There is a specific event triggering that re-render. But what if a re-render itself needs to sometimes connect externally? That is the case for useEffect.

    The details:

    It takes two arguments: useEffect(setup, dependencies?)

    setup: is a function that runs some code to say connect to the server.

    dependencies: is an optional array of variables to check for changes. If useEffect causes a state change, then then causes a rerender that will run the useEffect again, so we need a way to prevent this and on rerun useEffect if necessary. Say when a state or props variable in the dependency array changes.

    and useEffect returns, option cleanup code. Woof!

Debouncing

You may notice that it searches immediately when you type. This both desired, but also it seems a little laggy. That is because for every keystroke it immediately launches an asynchronous web API request. This isnâ€™t quite ideal. You might want to issue a fewer requests. The technique for this is called debouncing. The term comes from hardware input where buttons will sometimes bounce slightly â€” enough to trigger a few keypresses where only one was really intended. Debouncing is simple, if a function was called, donâ€™t let it be called again within some small timeout. Weâ€™re going to use lodash.debounce to help us.

lodash.debounce is a higher order function. It takes as an argument a function and a timeout and returns a new function that is debounced.

ğŸ’¡ The useCallback hook is structured similarly to useEffect. It returns a memoized version of the callback that is passedâ€”in, in this case debounce(search,500). The debounce function returns a debounced version of the search function that batches up calls every half-second (500 milliseconds). This hook, similar to useEffect, is useful for preventing unnecessary renders, but specifically for callback functions. Since a hooks based component is just an function, every const and function definition is recreated every render call - to prevent that, useCallback takes a function and returns a version of it that persists across renders.

It memorizes the function!
