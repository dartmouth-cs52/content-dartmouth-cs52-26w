HTML & Basic CSS

Start with coding up a single question and answer - then use flexboxes to arrange them. At this point using flexboxes for this should be almost second nature. You don‚Äôt need any of those fancy grid frameworks.

One thing you may have noticed is that when you select an answer it is a exclusive selection. There is an input type that already does that automatically. Yup, radio boxes!

But the default styling of radio boxes is so ugly! Sure, but you can do something similar to the checkbox hack and hide the actual radio box but use it for some display logic.

Here is the basic syntax for a radio button with an image label.

HTML
12
<label for="puppy_question"><img src="http://onlines/someimage.png" /></label>
<input type="radio" id="puppy_question" value="puppy" />
CSS
123
#puppy_question {
  display: none;
}

You can actually nest the input inside of the label and then you don‚Äôt have to do the for="elementID" part of the label, it will just automatically serve as the label for the input inside it.

HTML
1234
<label>
  <img src="http://onlines/someimage.png" />
  <input type="radio" value="puppy" />
</label>

And you can suppress all radio inputs like so:

CSS
123
input[type='radio'] {
  display: none;
}

But how do you style the different states? Remember, the specs require 4 states: initial, hover, selected, and not-selected-when-another-is-chosen. The first two are straightforward:

CSS
123456789101112
/* 1. Initial state - style the label */

  border: 2px solid #ccc;
  cursor: pointer;
  transition: all 0.2s ease;
}

/* 2. Hover effect */
label:hover {
  border-color: #007bff;
  transform: scale(1.02);
}

States 3 & 4: Selected and Unselected Styling

States 3 (highlight selected) and 4 (dim unselected) are trickier with nested inputs. CSS traditionally can't "look up" the DOM tree to style a label based on its child input's state. Two approaches:

Option 1: JavaScript class toggle

Manage selection state entirely through classes ‚Äî the radio button just stores the value. Note: .closest() traverses up the DOM to find the first ancestor matching the selector.

JavaScript
12345
$('label').on('click', function() {
  const $question = $(this).closest('.question');
  $question.find('label').removeClass('selected');
  $(this).addClass('selected');
});

Option 2: CSS :has() selector

Modern CSS has :has() which selects elements based on their children. Here's state 3 (selected):

CSS
1234
/* Label containing a checked radio */
label:has(input:checked) {
  border-color: #28a745;
}

For state 4 (dim unselected), you need to target labels that are inside a question with a selection, but don't contain the checked input. Hint:

CSS
1234
/* Question has a selection, but this label doesn't have it */
.question:has(input:checked) label:not(/* what goes here? */) {
  opacity: 0.5;
}

Grouping Radio Buttons

The way you group radio buttons together (so clicking one de-selects the others) is via the name attribute. Radio buttons with the same name are mutually exclusive.

For example, these two radio buttons could both be selected at the same time (you can have a favorite dog and a favorite cat):

HTML
12345678
<label>
  <img src="http://onlines/someimage.png" />
  <input type="radio" name="favorite_dog" value="dalmation" />
</label>
<label>
  <img src="http://onlines/someimage.png" />
  <input type="radio" name="favorite_cat" value="tiger" />
</label>

While these two radio buttons will automatically unselect one when the other is clicked (you cannot have two favorite dogs):

</label>

Javascript Logic

Once you have the basic structure up you can dive into getting the logic working. At this point you should have some radio boxes on the screen that are checkable. But how do you collect the input from them?

Most likely you have a button that you would like to attach a javascript action to. Here's a tiny bit of code to get you started:

JavaScript
1234567891011
$('#idOfYourButton').on('click', function (e) {
  // gather all checked radio-button values
  var choices = $("input[type='radio']:checked")
    .map(function (i, radio) {
      return $(radio).val();
    })
    .toArray();
  // now you have an array of choices = ["valueofradiobox1", "valueofradiobox2", "valueofradiobox2"]
  // you'll need to do some calculations with this
  // a naive approach would be to just choose the most common option - seems reasonable
});

Feature Branches and Git Flow

Now that you have the basic quiz working, you'll add two more features using git feature flow. Both are required and each will be developed on its own branch.

We're going to start learning how to use github branches and pull requests for something called git feature flow.

Git Feature Flow is a branching strategy that helps teams manage feature development in a structured way. We aren‚Äôt yet working in a team, but we‚Äôre going to start using feature branches to keep track of our work.

In Git Feature Flow, you would normally:

  • git checkout -b my_new_feature create a new branch
  • work on branch, git commit, etc
  • git push origin my_new_feature push your new branch to github
  • go to github and create pull request comparing main branch to my_new_feature
  • (optionally) tag someone to review
  • (once they approve) - go back on GitHub and merge it in

This workflow helps keep the main branch stable while allowing parallel feature development. It also provides a nice history for your project with a distinct timeline for when certain features were added.

This is also how you might work with AI coding agents in the future ‚Äî assigning them a feature branch to work on while you work on another, then reviewing their PR before merging.

üí∞ Most likely you'll want to take your already working hardcoded HTML, abstract out some section, and output it from a javascript loop that constructs a big string of all the html you need and then append it to some element in the page.

You may find it irritating to do string concatenation like "<div>"+question.title+"</div>". Instead, use template literals (backticks) which let you embed variables directly:

JavaScript
1234567891011121314151617
$.getJSON('data.json', function(data) {
  let questionsHTML = '';

  // Loop through each question
  data.questions.forEach(function(question, index) {
    questionsHTML += `
      <div class="question">
        <h2>${question.question_name}</h2>
        <img src="${question.question_img_url}" />
        <!-- you'd loop through question.answers here too -->
      </div>
    `;
  });

  // Append all the generated HTML to a container
  $('#quiz-container').html(questionsHTML);
});
‚ö†Ô∏èAsync Timing: Notice that all your code must be inside the callback function. Code outside runs before the JSON loads! This is a common source of bugs ‚Äî if your quiz isn't showing up, check if your code is inside the callback.
