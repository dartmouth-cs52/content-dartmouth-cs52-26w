Media query

Now for the tricky part! Shrink the width of your browser window all the way down! GASP! üí©

Most likely it did not resize well. We‚Äôll deal with this using a single media query.

üöÄHere‚Äôs how to start:

CSS
12
@media only screen and (max-width: 640px) {
}

Anything in this media query will only apply if the screen is fairly narrow. Don‚Äôt worry about it working on every single device - rather focus on it working in a more general fashion in a ‚Äúmobile‚Äù width vs ‚Äúdesktop‚Äù. Flex-boxes will take care of things being responsive in most widths but you want to make sure that there is a mobile version where a few bigger site wide changes happen: rows into columns, nav links into hamburger menu.

‚ö†Ô∏è‚ö†Ô∏è‚ö†Ô∏è Add <meta name="viewport" content="width=device-width, initial-scale=1"> to the head section of your html. This tells the mobile browser to not pretend to be a real browser and use actual device pixels for width and allows you to test on mobile / in the Chrome mobile device mode.

Important Techniques

CSS
1
flex-direction: column;

üöÄ¬†Take some of your row flex boxes and simply convert them to columns. This works remarkably well for many cases. The input box and button, the links at the bottom, the highlight selections in the middle.

CSS
1
display: none;

This method can be very powerful. For your navbar the recommendation is to do this:

  1. create 2 separate nav elements.
    1. <nav class="desktop">
    2. <nav class="mobile">
  2. Then in your stylesheet you can do:
    1. by default nav.mobile { display: none;}
    2. and then in your @media query
    3. switch which is displayed and which is hidden!
    4. nav.mobile { display: flex;} & nav.desktop { display: none;}

This way you can toggle the display property on completely different sections of the site. You might have a completely different set of elements for the links section for instance. Toggle one off and the other on.
