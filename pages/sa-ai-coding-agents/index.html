What is Claude Code?

Claude Code is an agentic coding tool that runs in your terminal. Unlike the web chat interface, Claude Code can read and edit files in your project, run commands, and help you build things directly in your codebase.

Here's what it looks like when you run claude in a project directory:

Terminal
$ cd my-project
$ claude

Part 1: Configure Your Learning Agent

The starter repo includes a CLAUDE.md file. This is a special file that Claude reads at the start of every conversation ‚Äî it's like giving Claude instructions about how to work with you.

Open it and take a look. It tells Claude to:

  • Explain concepts before and after writing code
  • Use analogies to CSS selectors you already know
  • Focus on teaching, not just producing code
  • Check in to make sure you understand
üí¨Feel free to customize the CLAUDE.md! Want Claude to be more concise? Add that. Prefer more examples? Tell it. This is your learning agent.

Part 2: Learn jQuery with Claude

Now for the fun part! Open Claude Code in your cloned repo:

Terminal
cd your-repo-name
claude

Try these prompts in order. Claude will build something with you while explaining each concept:

Prompt 1: Getting Started

üí¨"I'm new to JavaScript and jQuery. Walk me through creating a simple HTML page with a button. When I click the button, it should add a new item to a list. Explain each concept as you go."

Prompt 2: Understanding Selectors

üí¨"Now help me understand how jQuery selectors work. Show me how they're similar to CSS selectors I already know."

Prompt 3: Event Handling

üí¨"Add a feature where clicking on any list item removes it. Explain how event handling works."

Prompt 4: Loading Data (if time)

üí¨"How would I load data from a JSON file and display it on the page? This is prep for my quiz assignment."

Tips for Working with Claude Code

  • Use plan mode - Type /plan or ask Claude to "plan this out first" before building. This helps you understand the approach before code gets written.
  • Ask questions! - If Claude writes something you don't understand, stop and ask. That's the whole point.
  • Use CLAUDE.md for memory - If you want Claude to remember something for future sessions, ask it to add notes to CLAUDE.md.
  • Test in browser - If you set up Chrome integration, ask Claude to open your page and test it.

What You Should Learn

By the end of this activity, you should understand:

Concept What it means
$() jQuery's way of selecting elements ‚Äî works just like CSS selectors
.append() Add new content inside an element
.on('click', ...) Run code when something is clicked
.text() / .html() Get or change the content of an element
.addClass() / .toggleClass() Add or toggle CSS classes on elements ‚Äî great for selected states
DOM manipulation Changing the page after it loads ‚Äî this is what makes sites interactive!
These are the same concepts you'll use in Lab 2 to build your Buzzfeed-style quiz. The quiz loads questions, displays them dynamically, and responds to user clicks ‚Äî all using jQuery!
