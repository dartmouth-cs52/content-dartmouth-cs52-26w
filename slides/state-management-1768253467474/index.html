Zustand
 

A small, fast and scalable bearbones state-management solution using simplified flux principles. Has a comfy API based on hooks, isn't boilerplatey or opinionated.

* it is unopinionated, hence we'll have to bring our own opinions!

Why zustand over redux?

  • Simple and un-opinionated
  • Makes hooks the primary means of consuming state
  • Doesn't wrap your app in context providers
  • Can inform components transiently (without causing render)

 

 
main premises
 
* this last one mostly means that it can operate outside of react if you want to, we most likely never will but for very fast paced updates for instance where you might have say a game loop triggering re-renders anyway, but mouse events happening much faster you can update state and not trigger a cascade of re-rendering
 
Step 1: create a store
 
import { create } from 'zustand'

const useBearStore = create((set) => ({
  bears: 0,
  increasePopulation: () => set((state) => ({ bears: state.bears + 1 })),
  removeAllBears: () => set({ bears: 0 }),
}))
 
Step 2: use the store hook to filter state
 
function BearCounter() {
  const bears = useBearStore((state) => state.bears)
  return <h1>{bears} around here ...</h1>
}

function Controls() {
  const increasePopulation = useBearStore((state) => state.increasePopulation)
  return <button onClick={increasePopulation}>one up</button>
}
 
Immer middleware for immutability
 
const useBearStore = create((set) => ({
  bears: 0,
  increasePopulation: () => set((state) => ({ bears: state.bears + 1 })),
}))


const useBearStore = create(immer(set) => ({
  bears: 0,
  increasePopulation: () => set((draft) => { draft.bears += 1 }),
})))

ie: use immer and like it

* immer middleware - no need for produce
