    • but React does it in O(n)O(n)

  • reconciliation heuristic

  • assumption 1: if elements are of different types, subtrees likely different

  • assumption 2: use unique key on each node (like a hashvalue) as stability hint/identifier

tree diffs
 
* heuristic O(n) algorithm * assumption1: Two elements of different types will produce different trees. thus subtrees will differ * 2: developer can hint at which child elements may be stable across different renders with a key prop. * https://reactjs.org/docs/reconciliation.html
