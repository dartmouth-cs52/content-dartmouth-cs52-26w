flexbox model
 
* **justify is main** * **align is cross** * default is row * *main axis* / *main dimension*: * primary axis along which flex items are laid out * extends in the *main dimension*. * *main-start* / *main-end* * flex items are placed starting on the *main-start* side * and going toward the *main-end* side. * *main size* / *main size property* * flex items width or height, whichever is in the main dimension, is the item’s main size * The flex item’s main size property is either the width or height property, whichever is in the main dimension. * *cross axis* / *cross dimension* * The axis perpendicular to the main axis is called the cross axis * extends in the cross dimension. * *cross-start* / *cross-end* * Flex lines are filled with items and placed into the container starting on the cross-start side of the flex container and going toward the cross-end side. * *cross size* / *cross size property* * The width or height of a flex item, whichever is in the cross dimension, is the item’s cross size. The cross size property is whichever of width or height that is in the cross dimension.

CSS Grid
 
Flex boxes aren't the only way for developers to build out structured pages. CSS grid is another viable alternative that can help build highly-structured experiences. The biggest difference between CSS grids and flex boxes is that CSS grids are two-dimensional, meaning the container's children elements are arranged in a grid, kind of reminiscent of graph paper. While flexbox is one-dimensional, there's only one direction of flow at a time for the flexbox (either row or column) but with overflow.
 
CSS Grids come to the rescue!
 
To further understand the difference between CSS grids and flexboxes, I've created a Code Sandbox with a basic CSS grid. We're going to use this sandbox to explore core concepts for CSS grid.
  • two dimensional grid with cells that can span specified widths and lengths
 
CSS Grid model
 
Setting up a CSS grid starts with the same process of identifying the container parent that will have its display property changed. Rather than flex, in this case, we want to use the grid display value. This change will then allow all children elements to use other CSS grid attributes to position themselves within the grid. Ie with flex - you can say it is a row and align cross direction but with grid you can go in both direction
  • container is the wrapping element that displays the grid
  • grid item is the child that displays on the grid 
<div class="container">
  <div class="item" id="item1"> </div>
  <div class="item" id="item2"> </div>
  <div class="item" id="item3"> </div>
</div>
.container {
  display: grid;
}
 
grid container & grid item
 
Element grid item has the potential to change in size and position when using grid css rules.
 
grid-area, column/row start/end
 
.item-d {
  /* grid-area: <name> | <row-start> / <column-start> / <row-end> / <column-end>; */
  grid-area: 1 / 4 / 4 / 6;
}
// OR
.item-d {
  grid-column-start: 4;
  grid-column-end: 6;
  grid-row-start: 1;
  grid-row-end: 4;
}
more: https://css-tricks.com/snippets/css/complete-guide-grid/ grid-area is a CSS property that takes four coordinates, the start row coordinate, then the start column coordinate, then the end row coordinate, and then the end column coordinate. By specifying these slash-separated numbers, CSS grid is able to calculate the surface area the grid item is supposed to take up within the grid. I like to think about each number as a start and end x/y coordinate. If any of the numbers exceed the specified number of rows and columns, the grid will currently stretch to fit that newly expanded content.
.container {

  row-gap: <line-size>;
  gap: <line-size>;

  /* old */

  grid-row-gap: <line-size>;
}
 
column-gap and row-gap
 
The column and row gap allow you to specify how much space should be present between each grid item along the rows and columns. If you want to specify the same gap space for both rows and columns you can use the shorthand gap.

transitions model

  • property: (required) of the element to apply to. can be specific property like background-color, a list, or all.

  • timing-function: (optional) animation interpolation type: linear, ease (default), ease-in, ease-out, ease-in-out, even cubic-bezier.
